# 古籍在线编辑系统（作业一实现）

本项目根据 `作业一说明.docx` 和 `XML-V0.1.docx` 实现了一个基于 HTML5 + JavaScript 的古籍在线编辑器。

## 1. 功能覆盖

### 1.1 扫描页导入

- 支持多张图片一次导入（按页形成页面序列）。
- 支持导入 PDF 文件，自动将 PDF 每一页渲染为图片后加入页面序列。
- 支持从工作区示例目录快速加载示例图片。

### 1.2 在线字/句/段选定与标注

- 在页面画布上拖拽即可创建标注区域。
- 支持三个层级：字、句、段。
- 编辑器左侧提供索引栏：
  - **标题树形索引**：支持大标题/子标题/子子标题/四级标题，可展开/折叠查看层级结构
  - 标题添加（可绑定当前选中标注，实现精准跳转）
  - **拖放功能**：支持将标题拖动到其他标题上来建立从属关系（创建层级结构）
- 页面画布支持交互浏览：
  - 鼠标滚轮缩放
  - 按住 Shift 拖拽平移（可左右移动页面）
- 支持三种标记方式：
  - 颜色高亮
  - 画框
  - 划线
- 支持右侧属性编辑：原文、简体、注释、注释类型、Unicode 编码、坐标等。

### 1.3 造字页面

- 支持录入无简体对应古字：
  - 自定义 Unicode 编码（例如 `U+E001`）
  - 上传字样图片
  - 从“编辑器”当前选中的标注框直接截取字样图片（无需本地上传）
  - 记录字样说明和备注
- 标注属性支持关联造字条目，自动回填编码。

### 1.4 XML 输出

- 生成并预览 XML 文本。
- 支持下载 XML 文件。
- 输出结构包含：
  - `article`
  - `head`
  - `content`（采用 `page_mode="1"` 单页模式）
  - `view`（按页生成 `svg`、`image`、标记图形与文字）
  - `sources`

## 2. 运行方式

当前版本为 前端 + Node.js + SQLite 持久化架构，建议使用 Node 服务启动。

### 2.1 环境准备

1. 安装 Node.js 18+。
2. 在项目根目录执行：

```bash
npm install
```

### 2.2 启动

```bash
npm start
```

默认访问地址：`http://localhost:3000`

- 页面入口：`/index.html`
- API 健康检查：`/api/health`

### 2.3 持久化位置

- SQLite 数据库：`data/sdudoc.sqlite`
- 扫描页图片：`uploads/pages/`
- 造字图片：`uploads/glyphs/`

> PDF 导入说明：前端仍通过 PDF.js（CDN）拆页，再上传到后端持久化。若网络受限导致 PDF.js 加载失败，可先改用图片导入。

## 3. 文件结构

- `index.html`：页面结构（编辑器 / 造字库 / XML 预览）
- `styles.css`：界面样式与响应式布局
- `app.js`：前端核心逻辑（画布标注、属性编辑、造字管理、调用后端 API）
- `server/index.js`：Node.js API 入口与静态资源服务
- `server/db.js`：SQLite 表结构、数据读写、图片落盘逻辑
- `server/xml.js`：服务端 XML 生成器

## 4. 与 XML-V0.1 的对应说明

- 标注区域在 `content` 中映射为 `paragraph -> sentence -> word -> char`。
- 每条标注生成唯一 `char id`，并在 `view/svg/text` 中使用同一 `id` 对应。
- 页面原图路径在 `sources/source` 与 `view/svg/image` 中同步记录。
- 标注图形：
  - 高亮/画框使用 `rect`
  - 划线使用 `line`

## 5. 已知简化点

- 当前实现默认采用 `content page_mode="1"`（单页模式），未实现 `text` 与 `div` 混合模式编辑器。
- 当前为单用户模式，尚未实现登录鉴权与多人并发冲突处理。
- 当前版本管理为“手动保存 XML 版本”，未实现自动增量版本。

## 6. 已实现扩展能力

- Node.js + SQLite 持久化（扫描页图片、标注、造字、XML 历史版本）。
- 服务端生成 XML 并支持手动保存版本。
- 页面刷新或服务重启后可恢复数据。

## 7. 后续可扩展方向

- 多人协同编辑与审校流程（登录、权限、乐观锁）。
- OCR/版面分析辅助自动框选与文本预填。
- 大型 PDF 的异步任务化导入与进度管理。
